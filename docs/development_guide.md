## 1. はじめに：全体の流れを知ろう

### 🔤 最初に知っておくべき用語

| 用語 | 意味 | たとえると… |
|------|------|-----------|
| **Git（ギット）** | ファイルの変更履歴を記録するシステム | 「無限にやり直しができるセーブシステム」 |
| **GitHub（ギットハブ）** | Gitで管理したファイルをネット上で共有するサービス | 「みんなで使えるクラウド上のセーブデータ置き場」 |
| **リポジトリ** | プロジェクトのファイルを保管する場所 | 「プロジェクト専用のフォルダ」 |
| **Clone（クローン）** | GitHubからファイルを自分のPCにダウンロードすること | 「セーブデータのコピーをもらうこと」 |
| **Pull（プル）** | GitHubの最新情報を自分のPCに取り込むこと | 「最新のセーブデータに更新すること」 |
| **Commit（コミット）** | 自分の変更を「セーブ」すること | 「チェックポイントを作ること」 |
| **Push（プッシュ）** | セーブしたデータをGitHubにアップロードすること | 「セーブデータをクラウドに送ること」 |
| **Antigravity（アンチグラビティ）** | AIがコードを書いたり調べたりしてくれる開発ツール | 「超優秀なAIアシスタント付きのエディタ」 |
| **エディタ** | コード（プログラムの文章）を書くためのソフト | 「プログラミング専用のWordのようなもの」 |

### 開発の全体像

```
┌─────────────────────────────────────────────────────────┐
│                   開発の全体イメージ                      │
│                                                         │
│  あなたのPC                    GitHub（ネット上）          │
│  ┌────────────┐    Push →     ┌──────────────┐          │
│  │ V EFFECT   │ ──────────→  │  V EFFECT    │          │
│  │ フォルダ    │ ←──────────  │  リポジトリ   │          │
│  └────────────┘    ← Pull     └──────────────┘          │
│        ↕                            ↕                   │
│   Antigravityで               パートナーのPC             │
│   コードを作成             ┌────────────┐               │
│                            │ V EFFECT   │               │
│                            │ フォルダ    │               │
│                            └────────────┘               │
└─────────────────────────────────────────────────────────┘
```

## 2. 開発環境の構築

### ステップ1：必要なものをインストールする

以下の3つをパソコンにインストールしてください。

#### ① Git（ファイル管理システム）

1. [https://git-scm.com/downloads](https://git-scm.com/downloads) にアクセス
2. 自分のOS（Windows / Mac）に合ったものをダウンロード
3. インストーラーを実行（基本的にすべて初期設定のままで「Next」を押し続ければOK）
4. インストール完了後、ターミナルで以下を確認：
   ```
   git --version
   ```
   → バージョン番号が表示されれば成功！

#### ② Antigravity（AI開発アシスタント）

1. Antigravityのインストーラーを実行するか、案内されたURLからダウンロード
2. インストール後、起動して初期設定を行う
3. AIモデル（Gemini / Claude）のどちらかが使えることを確認
4. 拡張機能（左のアイコン） > Japanese Language Pack for Visual Studio Code」検索 > インストール > 再起動　で日本語化
    □◇
    □□

#### ③ Flutter SDK（アプリ開発フレームワーク）

1. 拡張機能 > Flutter 検索 > DartとFlutterをインストール
2. Ctrl＋Shift＋P > Flutter 入力 > Flutter: New Project 選択 > Download SDK > ダウンロード（c: を選んでそこにダウンロードするのがおすすめ）
3. インストール完了後、ターミナルで以下を実行して確認：
   ```
   flutter doctor
   ```
   → 各項目に ✓ マークが表示されれば成功！（✗ のところはチャットに投げて対応してもらう）
4. **Android Studio もインストール推奨**（Androidエミュレーター＝仮想スマホが使えるようになる）
   - [https://developer.android.com/studio](https://developer.android.com/studio) からダウンロード

> **💡 Flutter って何？**
> Google が作った「1つのプログラムで Android と iOS の両方のアプリが作れる」ツールです。
> Dart（ダート）というプログラミング言語を使います。
> 「**Hot Reload（ホットリロード）**」という機能があり、
> コードを変更すると**数秒で画面に反映される**ので、作りながらすぐ確認できます。

### ステップ2：GitHubアカウントを作る

1. [https://github.com/](https://github.com/) にアクセス
2. 「Sign up」から無料アカウントを作成
3. メールアドレスの認証を完了する

### ステップ3：プロジェクトのファイルをダウンロードする

Antigravityのターミナルで以下のコマンドを実行します。
（ダウンロード先は自分の分かりやすい場所に変更してOK）

```bash
# 自分のPCにプロジェクトをダウンロード
git clone https://github.com/YusukeKatahara/V EFFECT.git
```

> **💡 「ターミナル」って何？**
> Antigravity の画面下部にある、文字を打ち込む黒い画面のことです。
> ここにコマンド（命令文）を入力して Enter を押すと、パソコンに指示を出せます。
> でも心配しないでください。Antigravityに「クローンして」と言えば代わりにやってくれます。

### ステップ4：Gitの初期設定をする

Antigravity に以下のように話しかけてください（右にあるチャット欄）：

> 「Gitの初期設定をして。名前は〇〇、GitHubアカウント名は△△です」

AIが以下のようなコマンドを実行してくれます：
```bash
git config --global user.name "あなたの名前"
git config --global user.email "GitHubアカウント@users.noreply.github.com"
```

### ステップ5：セットアップガイドに記入する

1. `setup/setup_guide.md` を開く
2. 各項目を Antigravityと相談しながら記入する
3. 完成したら Antigravity に **「`/apply-setup` を実行して」** と伝える
4. AIが自動で開発環境の設定ファイルを最適化！

**🎉 これで環境構築は完了です！**

## 3. 日々の開発作業の流れ

### 基本のサイクル

毎日の開発は、以下の **4ステップ** を繰り返すだけです。

```
① Pull（最新化） → ② 開発 → ③ Commit（保存） → ④ Push（共有）
```

### 各ステップの詳しい説明

---

#### ① Pull（プル）— パートナーの更新を取り込む

**いつやる？** → 作業を始めるとき（毎回）

**Antigravity への言い方：**
> 「作業を始めるので、最新のコードを取ってきて」

**AIが実行するコマンド：**
```bash
git pull origin main
```

**何が起こる？**
- パートナーが前回アップロードした変更が、あなたのPCにも反映されます
- 「変更なし」と言われたら、すでに最新です。そのまま作業を始めてOK

---

#### ② 開発 — Antigravity と一緒にコードを書く

**Antigravity への言い方の例：**
```
「ログイン画面を作って」
「ボタンを押したら画面が切り替わるようにして」
「このエラーを修正して」
「ホーム画面のデザインを変えて。もっとシンプルにしたい」
```

**ポイント：**
- 「何をしたいか」を日本語で伝えるだけでOK
- 分からない専門用語があれば「〇〇って何？」と聞けばAIが説明してくれます
- 作業の途中でも「今何を変えたか教えて」と聞けば、変更の概要を教えてくれます

---

#### ③ Commit（コミット）— 変更を保存する

**いつやる？** → 1つの作業がキリのいいところまで終わったとき

**Antigravity への言い方：**
> 「今の変更を保存（コミット）して。XXXのデザインを作成しました」

**AIが実行するコマンド：**
```bash
git add -A
git commit -m "XXXのデザインを作成"
```

**コミットメッセージ（変更メモ）のコツ：**
| 良い例 ✅ | 悪い例 ❌ |
|----------|----------|
| XXXのボタンを追加 | 修正 |
| YYYの機能を実装 | 変更 |
| ZZZの色をブランドカラーに統一 | 更新 |

> **💡 ポイント：** 何を変えたかが後から分かるように書くのが大切です。
> 面倒なら「コミットして」とだけ言えば、AIが変更内容を見て適切なメッセージを自動作成してくれます。

---

#### ④ Push（プッシュ）— 変更をパートナーに共有する

**いつやる？** → コミットした後。特に作業を終えるとき

**Antigravity への言い方：**
> 「Pushして（共有して）」

**AIが実行するコマンド：**
```bash
git push origin main
```

**何が起こる？**
- あなたの変更がGitHubにアップロードされます
- パートナーが次に Pull したとき、あなたの変更が反映されます

---

### 1日の作業の例

```
🕐 作業開始
│
├─ 「最新のコードを取ってきて」（Pull）
│
├─ 「XXXを作って」（開発）
│
├─ 「今の変更をコミットして」（Commit）
│
├─ 「YYYも追加して」（開発の続き）
│
├─ 「今の変更もコミットして」（Commit）
│
├─ 「Pushして」（Push — まとめてアップロード）
│
🕔 作業終了
```

> **💡 コミットは何回やってもOK。Pushは最後にまとめて1回でも大丈夫です。**
> ただし、長時間Pushしないとパートナーとの変更が衝突しやすくなるので、
> できれば作業中にもこまめにPushすることをおすすめします。

## 4. よくあるトラブルと解決法

### 😱 「Pullしたらエラーが出た！」

**原因：** あなたとパートナーが同じファイルの同じ部分を変更した（＝コンフリクト／衝突）

**解決法：**
> Antigravity に「Pullでコンフリクト（衝突）が起きたので解決して」と伝えてください。
> AIがどちらの変更を採用するか、または両方を統合するか提案してくれます。

**予防法：**
- 事前に「今日はどこを触るか」をパートナーと共有しましょう
- 同じファイルを同時に編集しないように分担を決めましょう

---

### 😱 「間違えてファイルを消しちゃった！」

**解決法：**
> Antigravity に「〇〇.dart ファイルを間違って消してしまったので、前の状態に戻して」と伝えてください。

Gitは全ての変更を記録しているので、**過去のどの時点にも戻せます**。安心してください。

---

### 😱 「Pushしようとしたらエラーが出た！」

**原因：** パートナーが先にPushしていて、あなたのデータが古い

**解決法：**
> Antigravity に「Pushでエラーが出たので解決して」と伝えてください。
> 通常は「まずPullしてからPushし直す」ことで解決します。

---

### 😱 「AIが変なコードを書いた気がする…」

**解決法：**
1. Claude モデルに切り替えて「このファイルのコードをレビューして」と頼んでみましょう
2. AIの提案をまず読んで、分からなければ「もっと簡単に説明して」と言えばOK
3. 問題があれば「元に戻して」と言えば直前の状態に戻せます

## 5. リリースまでのロードマップ

### 全体の流れ

```
Phase 1        Phase 2         Phase 3        Phase 4        Phase 5
企画・設計  →  プロトタイプ  →  本開発     →  テスト     →  リリース
（何を作る？） （試作品）      （全機能作成）  （バグ修正）   （公開！）
```

### 各フェーズの詳しい内容

#### Phase 1：企画・設計 🎯

**やること：**
- [ ] アプリのコンセプト（どんな問題を解決するか）を決める
- [ ] ターゲットユーザー（誰が使うか）を決める
- [ ] 主要な機能の一覧を作る
- [ ] 画面のラフスケッチ（手描きでOK）を描く
- [ ] 技術スタック（どの技術で作るか）を決める
- [ ] `setup/setup_guide.md` を全員が記入して `/apply-setup` を実行

**Antigravityへの話しかけ方：**
> 「〇〇なアプリを作りたいんだけど、必要な機能を一覧にして」
> 「画面遷移図（どの画面からどの画面に移動できるか）を作って」

---

#### Phase 2：プロトタイプ（試作品） 🔨

**やること：**
- [ ] 主要な画面（3〜5画面）だけ見た目を作る
- [ ] 画面間の移動（ナビゲーション）を実装する
- [ ] 実際に触ってみて、使いやすさを確認する

**PC上でアプリを確認する方法：**
- `flutter run` コマンドで **Android エミュレーター**（PC上の仮想スマホ）に表示
- `flutter run -d chrome` で **ブラウザ上でもプレビュー可能**（手軽に確認したいとき）
- **Hot Reload** でコード変更が即座に画面に反映される（保存するだけでOK）
- USBケーブルで実機（自分のスマホ）を接続して確認することも可能

**ゴール：** 「こんな感じのアプリになるよ」と人に見せられる状態

---

#### Phase 3：本開発 💻

**やること：**
- [ ] 全ての機能を実装する
- [ ] データベース（情報の保存場所）を構築する
- [ ] ログイン機能など、セキュリティに関わる機能を実装する
- [ ] エラーハンドリング（エラー時の対処）を入れる

**ポイント：** 一度に全部作ろうとせず、1つずつ機能を完成させていくのがコツ

---

#### Phase 4：テスト 🧪

**やること：**
- [ ] 全ての機能が正しく動くか確認する
- [ ] 色々な端末サイズで表示が崩れないか確認する
- [ ] ネットワークが切れた時にアプリが落ちないか確認する
- [ ] 友人や知り合いに使ってもらい、感想をもらう（ベータテスト）

**Antigravityへの話しかけ方：**
> 「ログイン機能のテストケースを洗い出して」（Claude推奨）
> 「このコードにセキュリティの問題がないかチェックして」（Claude推奨）

---

#### Phase 5：リリース（公開） 🚀

**やること：**
- [ ] プライバシーポリシーを作成する
- [ ] 利用規約を作成する
- [ ] ストア用のスクリーンショットを撮影する
- [ ] アプリの説明文を書く
- [ ] Apple Developer / Google Play のアカウントを作成する
- [ ] 審査に提出する
- [ ] 審査でリジェクト（不合格）されたら修正して再提出

> **⚠️ 重要：** リリース時のリスクや注意点の詳細は `docs/release_risk_guide.md` を参照してください。

## 6. 進捗の確認方法

### 方法1：Antigravity に聞く

> 「今のプロジェクトの進捗状況を教えて」
> 「最近どんなファイルが変更された？」
> 「まだ完成していない機能はある？」

### 方法2：GitHub で確認する

ブラウザで [https://github.com/YusukeKatahara/V EFFECT](https://github.com/YusukeKatahara/V EFFECT) にアクセスすると：
- **最新のファイル一覧** が見られます
- **誰が・いつ・何を変更したか** の履歴（Commitsタブ）が見られます
- パートナーの作業状況も確認できます

### 方法3：Git コマンドで確認する

Antigravity に以下のように頼むこともできます：

| 知りたいこと | Antigravityへの話しかけ方 |
|-------------|-------------------------|
| 最近の変更履歴 | 「最近のコミット履歴を見せて」 |
| 今の変更点 | 「今、何が変更されているか教えて」 |
| パートナーの最新の変更 | 「Pull して、何が更新されたか教えて」 |
